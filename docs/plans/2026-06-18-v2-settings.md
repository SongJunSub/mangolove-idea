# Settings UI (V2 item E) â€” Implementation Plan

For agentic workers: **REQUIRED SUB-SKILL: superpowers:subagent-driven-development**

## Goal

Productize the three command "env seams" â€” agent command, server command, verify command â€” plus the merge/diff **base branch**, which today are hidden dev/test-only environment variables (`MANGO_AGENT_CMD`, `MANGO_SERVER_CMD`, `MANGO_VERIFY_CMD`) and a hardcoded `'main'`. After this feature a tiny `SettingsStore` persists a per-project `AppSettings` JSON, a **gear (âš™) button** in the toolbar opens a **Settings modal** to view/edit those four values, and the four resolution points in `register-ipc.ts` resolve each as **settings value (if set) > existing env seam > hardcoded default**. Because settings default to **unset (undefined)**, the existing Playwright smokes (which set the env seams with no persisted settings) KEEP WORKING â€” env still wins when settings are unset. This is **additive only**: no existing contract type, channel, or manager class is reshaped.

## Architecture

Mirrors the Plan-5 `SessionStore` + the `register-ipc` lazy-getter/injected-emitter seams exactly:

- **Main â€” persistence**: a new `SettingsStore` (mirrors `SessionStore` field-for-field): persists `AppSettings` to an **injected** path (default `app.getPath('userData')/'settings.json'`), corrupt-safe (bad/missing/non-object â†’ `{}`), sanitized to ONLY the 4 known **string** fields (drops unknown keys, ignores non-strings), atomic temp+rename write. Constructed **eagerly** in `index.ts` (holds the real electron `app` for the userData path) and assigned to `ctx.settingsStore` BEFORE `registerIpc`, so `getSettingsStore(ctx)` reads it **synchronously** (exactly like `getSessionStore`). Unit-tested against a temp file.
- **Main â€” precedence wiring (the core)**: the four resolution points in `register-ipc.ts` read `getSettingsStore(ctx).get()` and resolve `settings ?? env ?? default`. A small pure exported helper `resolveCommands(settings)` makes the precedence unit-testable without Electron.
- **Main â€” live-apply (orphan-safe)**: a new `SETTINGS_SET` handler persists, then **clears the cached managers that bake the command/base in at construction** so they lazily rebuild with the new settings on next use. The stateless `mergeRunner`/`diffViewer` (no live OS process) are **always** cleared â€” so an edited `verifyCommand`/`baseBranch` applies on the very next merge/diff. But `sessionManager`/`serverManager` own **live child processes** that the before-quit kill-sweep (`index.ts`) and the live `SESSION_INPUT`/`RESIZE` handlers find THROUGH `ctx`; nulling them mid-session would orphan the running `claude`/server from the sweep AND detach it from the next keystroke (a fresh manager has no record of it). So they are cleared **only when idle** (`sessionManager.liveWorktreeIds()` empty / `serverManager.hasLiveServer()` false); while busy they are kept, and a changed agent/server command takes effect once the live work ends (documented). Clearing only affects the production rebuild path â€” tests inject the managers directly and assert each branch.
- **IPC (additive)**: two new channels `SETTINGS_GET` / `SETTINGS_SET`, a new `AppSettings` type, a new `settings` namespace on `MangoApi`, preload wiring. Mirrors how Plan 5 (`SESSION_RECORDS`) and Monaco (`DIFF_*`) added channels additively. **No existing contract changes.**
- **Renderer**: a `useSettings` hook over `window.mango.settings`, and a Settings **modal** (mirrors the Plan-5 quit-warning modal in `App.tsx`: a fixed-overlay `role="dialog"` with buttons) opened by a gear button placed next to the existing `<Toolbar>`. Four text inputs seeded from `settings.get()`; **Save** calls `settings.set()` + closes; **Cancel** closes. ALL existing UI stays.
- **Verification**: vitest **node** tests for `SettingsStore` (temp file) + `resolveCommands` precedence + `SETTINGS_GET`/`SETTINGS_SET` delegation & manager-clearing (fake store), in the existing `ipc-roundtrip.test.ts` / `session-store.test.ts` style. The modal verified by typecheck/lint/build + a **documented (NOT committed)** Playwright smoke.

```ts
import { existsSync, readFileSync, renameSync, writeFileSync } from 'node:fs';
import { join } from 'node:path';
import type { AppSettings } from '../../shared/types';

/**
 * Resolves the default settings.json path under Electron's userData dir. Kept
 * separate (mirrors getDefaultSessionsPath) so register-ipc reads the store from
 * ctx while tests inject an explicit temp path into the SettingsStore constructor.
 */
export function getDefaultSettingsPath(getUserDataPath: () => string): string {
  return join(getUserDataPath(), 'settings.json');
}

/** The four known AppSettings keys â€” the ONLY keys ever read/written. */
const KNOWN_KEYS: readonly (keyof AppSettings)[] = [
  'agentCommand',
  'verifyCommand',
  'serverCommand',
  'baseBranch',
];

/**
 * Persists the V2 item E AppSettings to a single JSON file whose path is injected
 * (tests use a temp file). Mirrors SessionStore EXACTLY: never throws on a
 * missing/corrupt/non-object file (treated as empty {}), sanitizes to ONLY the
 * four known STRING fields (drops unknown keys, ignores non-strings), and writes
 * atomically (temp file + rename) so a crash mid-write cannot leave a half file.
 *
 * EVERY field is optional: an unset field means the caller falls back to the env
 * seam then the hardcoded default (precedence: settings > env > default).
 */
export class SettingsStore {
  private readonly filePath: string;

  constructor(filePath: string) {
    this.filePath = filePath;
  }

  /** Reads + parses settings, returning {} on missing/corrupt/non-object files. */
  load(): AppSettings {
    if (!existsSync(this.filePath)) return {};
    let parsed: unknown;
    try {
      parsed = JSON.parse(readFileSync(this.filePath, 'utf8'));
    } catch {
      return {}; // corrupt JSON -> recover as empty
    }
    return this.sanitize(parsed);
  }

  /** Alias for load() â€” present so callers reading "current settings" read clearly. */
  get(): AppSettings {
    return this.load();
  }

  /**
   * Merges a partial into the current settings, then persists. A key PRESENT in
   * `partial` with an empty string (or a non-string) is treated as a DELETE of that
   * key (reverts to env/default) â€” so clearing a field in the Settings modal really
   * unsets it instead of leaving the old persisted value stuck. A key ABSENT from
   * `partial` is left untouched (true partial-merge). Sanitized to the known fields.
   */
  set(partial: Partial<AppSettings>): AppSettings {
    const merged: Record<string, string> = { ...this.load() };
    const source = partial as Record<string, unknown>;
    for (const key of KNOWN_KEYS) {
      if (!(key in source)) continue; // not in this partial -> leave as-is
      const value = source[key];
      if (typeof value === 'string' && value !== '') {
        merged[key] = value;
      } else {
        delete merged[key]; // '' or non-string -> unset (revert to env/default)
      }
    }
    this.write(merged);
    return merged;
  }

  /** Projects an input to EXACTLY the four known STRING fields (drops anything else). */
  private sanitize(raw: unknown): AppSettings {
    if (raw === null || typeof raw !== 'object') return {};
    const source = raw as Record<string, unknown>;
    const out: Record<string, string> = {};
    for (const key of KNOWN_KEYS) {
      const value = source[key];
      if (typeof value === 'string') out[key] = value;
    }
    return out;
  }

  private write(settings: AppSettings): void {
    const tmp = `${this.filePath}.tmp`;
    writeFileSync(tmp, JSON.stringify(settings, null, 2));
    renameSync(tmp, this.filePath);
  }
}
```

```ts
import { useCallback, useEffect, useState } from 'react';
import type { AppSettings } from '../../shared/types';

/** Reads + writes the persisted AppSettings via window.mango.settings. */
export interface UseSettings {
  /** Current settings (empty until the initial fetch resolves). */
  readonly settings: AppSettings;
  /** True until the initial fetch resolves. */
  readonly loading: boolean;
  /** Persists a partial and updates local state with the merged result. */
  save(partial: Partial<AppSettings>): Promise<void>;
}

/** Fetches settings once on mount; save() persists a partial and refreshes state. */
export function useSettings(): UseSettings {
  const [settings, setSettings] = useState<AppSettings>({});
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    let alive = true;
    void window.mango.settings
      .get()
      .then((s) => {
        if (alive) setSettings(s);
      })
      .finally(() => {
        if (alive) setLoading(false);
      });
    return () => {
      alive = false;
    };
  }, []);

  const save = useCallback(async (partial: Partial<AppSettings>): Promise<void> => {
    const merged = await window.mango.settings.set(partial);
    setSettings(merged);
  }, []);

  return { settings, loading, save };
}
```

Mutating a live manager's private `command` would be invasive and reshape the manager classes (forbidden â€” additive only). A busy session/server keeps its old command until the live work ends â€” acceptable and documented.
- **Why this can't break the 144 existing tests**: the production getters are only reached when `ctx.<manager>` is undefined; every existing delegation test injects the manager, short-circuiting before `getSettingsStore`. The clearing in `SETTINGS_SET` runs only in the new settings tests (which inject `settingsStore` + managers whose idle/busy state they assert on). No existing channel, type, or class is modified â€” only additions (`hasLiveServer()` is a new public method, callers-add-only).
- **Base-branch validity is enforced downstream, not in the modal**: `settings.baseBranch` flows to both merge `targetBranch` and diff `base`. A typo'd/non-existent branch surfaces as a git error at merge/diff time (`DiffViewer.assertSafeRef` guards option-injection but not existence) â€” matching the existing renderer-supplies-base contract. Acceptable for an MVP settings pane.
- **The settings delegation test block intentionally needs no env save/restore**: only `resolveCommands` reads the `MANGO_*_CMD` env (and its own describe block already manages env in `beforeEach`/`afterEach`); the `SETTINGS_GET`/`SETTINGS_SET` handlers call only `getSettingsStore(ctx).get()/.set()`, never the env seams, so the settings block reads no ambient env.

'true'`, so when settings are unset the value passed is identical to the `MergeRunner` class's own default â€” behavior is unchanged for unset settings, and a settings value now wins.
